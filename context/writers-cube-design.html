Architecture & Design Overview · v0.5.0

Writer's Cube

A web app for novelists — brainstorming, outlining, drafting in a rich-text editor, organizing scenes/chapters on a visual canvas, and exporting a finished manuscript. Built as a server-rendered Next.js app with an AI-assisted writing layer.

TypeScript Next.js 16 · App Router React 19 (RSC) Supabase / Postgres Anthropic · claude-sonnet-4-5 Tailwind CSS v4

Languages

One typed language end-to-end, plus the supporting markup, styling, and schema languages.

Primary

TypeScript

All application code (strict mode). Shared types in src/lib/types.ts; client & server share the same models.

React 19 components — a mix of Server Components and "use client" islands for interactive surfaces.

CSS · Tailwind v4

Utility-first via the Tailwind v4 PostCSS pipeline, with a small globals.css.

How it's structured

A three-tier flow: a React client, a Next.js server-action data layer, and two external services. There is almost no REST surface — the UI calls typed Server Actions directly.

① Client — the browser React 19 · App Router
Tiptap (ProseMirror) — rich-text scene editor & manuscript scroll mode @xyflow/react — visual story canvas / mind-map dnd-kit — drag & drop reordering TanStack Query — fetching & cache Zustand — local UI state (appearance, tags, organize)
▼   typed Server Action calls   ▼
② Server — Next.js runtime "use server" · src/server/*
projects · scenes · characters · kernels outline · timeline · mindmap · canvas brainstorm · prompts import · publish · export · drive project-context (assembles AI context) + route handlers: /export, /auth/callback
▼   persistence & AI   ▼
③ External services via @supabase/ssr & @anthropic-ai/sdk
Supabase — Postgres data, Auth, per-project isolation (SSR client + middleware) Anthropic API — claude-sonnet-4-5 for brainstorm/AI assistance (server-side key) Google Drive — optional import/export integration

Feature modules

Draft

Rich-text editor

Tiptap with bold/italic/underline/strike, H1–H3, lists, quotes; a bubble toolbar; and a whole-manuscript continuous scroll mode.

Plan

Canvas & mind-map

React Flow board for arranging story kernels, scenes, and relationships; outline templates and a project timeline.

Ideate

AI brainstorm

Mode-driven prompts (brainstorm-modes.ts) sent to Claude with assembled project context; history saved per project.

Organize

Scenes, chapters & tags

Drag-and-drop ordering, loose vs. chaptered scenes, custom tag marks inside the prose, and tag extraction.

In

Import

.docx ingestion via mammoth; Google Drive file browser; paragraph-aware import into scenes.

Out

Publish & export

Manuscript export to docx and EPUB (jszip); publish settings; a separate Node/PDF build for briefs.

Design principles in play

Server Actions over REST

The data layer is typed functions marked "use server", called straight from components — fewer endpoints, end-to-end type safety, no client/API drift.

Per-project isolation

Most tables carry a project_id (see migrations 0018–0020); Story Bible, organize state, tags, and brainstorm are scoped per project.

AI as a server capability

The Anthropic key lives only on the server; project-context.ts builds the model's context so prompts stay grounded in the user's story.

Content as portable Markdown

Editor JSON converts to Markdown for storage/export, keeping the manuscript format-independent and round-trippable.
